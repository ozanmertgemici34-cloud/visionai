# Public Roadmap

This roadmap is written for a public showcase. It describes direction without revealing private implementation details.

## Current Preview

- Desktop assistant concept.
- Turkish-first interaction style.
- Persistent memory concept.
- Voice interaction concept.
- Tool-aware assistant flow.
- Holographic UI direction.

## Near-Term Goals

- Add safe demo screenshots.
- Add short UI preview GIF.
- Publish a high-level architecture diagram.
- Add memory system explanation.
- Add privacy and safety notes.
- Add a feature comparison table.

## Product Goals

- Safer desktop action permissions.
- Better memory review controls.
- Semantic memory recall.
- Project-scoped memory.
- Optional offline speech-to-text.
- Improved installer and startup flow.
- More polished QML visual states.

## Memory Goals

- Memory source tracking.
- Confidence scoring.
- User-editable memory review.
- Controlled merge and summarization.
- Better handling for outdated or conflicting memories.

## Safety Goals

- Confirmation flow for high-risk actions.
- Clear permission levels.
- Local-only sensitive data policy.
- Audit log for tool actions.
- User-visible explanation for actions.

## Showcase Goals

- Keep full source private.
- Share enough to prove the concept.
- Avoid publishing dangerous automation code.
- Avoid publishing personal data.
- Make the project look serious, clean, and investor/developer friendly.
